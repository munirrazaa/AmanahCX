How super admin allocates modules to a tenant workspace

  

    

      
1

      
Create Workspace

      
Super admin opens Workspaces → New Workspace. Enters company name, slug, plan tier and admin email.

    

    

      
2

      
Select Modules

      
A checklist of modules from MODULE_CATALOG is shown. Super admin ticks which ones the customer has paid for. Core CRM is always locked on.

    

    

      
3

      
Select Features

      
Within each module, individual features can be toggled. E.g. Sales module without Invoice Templates if not in the contract.

    

    

      
4

      
Tenant Sees Only What's Paid

      
Sidebar, API routes and role screens for that tenant show only entitled modules. Attempts to access unentitled routes return NOT_LICENSED 403.

    

  


  
What's built vs what's still needed

Sidebar filtered by entitlement
modules.ts NAV_FEATURE_MAP — all 7 modules mapped

    
Sales routes gated
requireEntitlement() guard — sales.invoices, sales.contacts etc.

    
Ticketing routes gated
requireEntitlement('ticketing.tickets','ticketing.sla','ticketing.csat')

    
Voice Bot routes gated
requireEntitlement('voice_bot.calls','voice_bot.config') — webhooks excluded

    
Email Inbox routes gated
requireEntitlement('emails.inbox','emails.compose')

    
Edit entitlements after creation
PUT /super-admin/tenants/:id/modules endpoint exists

    
Roles screen filtered per module
Sales done. Ticketing/Voice/Email role rows not yet hidden when module is off.

    
Subscription billing automation
No Stripe/payment gateway. Module upgrades are manual today — contract + super admin toggle.

  







  
Module dependency chain — what needs what

  


    

      
Core CRM

      →
      
Ticketing & Support

      →
      
Voice Bot

      Every ticket must link to a CRM contact. Voice Bot auto-creates the contact then raises the ticket.
    


    

      
Core CRM

      →
      
Sales & Invoicing

      Billing contacts are linked to CRM contacts. Invoice history appears on Contact 360.
    


    

      
Core CRM

      →
      
Email Inbox

      Incoming emails auto-match to CRM contacts by email address.
    


    

      
Analytics

      Sits on top of any combination. No dependency — but gets richer with every active module.
    


    

      
Integrations

      Horizontal. Required to configure Voice Bot provider (Retell/Vapi webhook setup).
    


    

      
Standalone vs. combo selling rules

      

        

          
✅ Can sell standalone

          

            • Core CRM alone (Free plan)

            • Core CRM + Ticketing

            • Core CRM + Email Inbox

            • Core CRM + Sales & Invoicing
          

        

        

          
⚡ Must bundle

          

            • Voice Bot needs Ticketing (and Core CRM)

            • Analytics needs at least one data module

            • Integrations should bundle with Voice Bot

            • Sales contacts link requires Core CRM contacts
